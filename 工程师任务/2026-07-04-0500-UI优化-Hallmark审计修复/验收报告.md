## 技术成果

### 1. Critical 修复（T1）

**问题 1：零 chroma 中性色**
- **修复前：** 9 个中性色 chroma 0.004–0.012，深色模式显得平坦
- **修复后：** 提升至 0.015–0.025，保持 260° 冷蓝锚定，增强立体感
- **文件：** `ui/src/styles/tokens.css`

**问题 2：单字体页面**
- **修复前：** 只有 JetBrains Mono，无字体配对
- **修复后：** 引入 Inter Tight Variable 作为 display 字体
  - **Display（Inter Tight）：** TopBar 品牌、运行中指标、卡片标题、状态标签
  - **Body（JetBrains Mono）：** 数值、路径、时间戳、代码
- **文件：** `tokens.css`, `index.css`, `TopBar.tsx`, `TaskCard.tsx`, `DetailPanel.tsx`
- **新增依赖：** `@fontsource-variable/inter-tight`

**问题 3：N9+Ft2 过于安静**
- **修复前：** Nav N9（edge-aligned minimal）+ Footer Ft2（inline muted）
- **修复后：** Nav N6（split-level command bar）+ Footer Ft6（stat bar）
  - **TopBar：** 左品牌 | 中「● N 运行中」| 右「在线 + 今日 N」
  - **StatusBar：** 左「~/.grok-acp · 保留 7 天」| 右「N 运行 · N 今日 · v0.2.0」
- **文件：** `TopBar.tsx`, `StatusBar.tsx`, `App.tsx`

### 2. Major 增强（T2-T5）

**T2：Running 卡片视觉增强**
- **padding：** Running 卡片 `p-lg`（24px）vs Compact 的 `p-md`（16px）
- **背景：** Running 使用 `--color-paper-3` vs Compact 的 `--color-paper-2`
- **效果：** Running 卡片在网格中"跳出来"，视觉更活跃
- **文件：** `TaskCard.tsx`

**T2：DetailPanel 动效增强**
- **打开：** `y: 12 → 0` 叠加 `scale: 0.96 → 1.0`
- **关闭：** `y: 0 → 12`（对称）
- **效果：** 从"中心弹出"改为"从下浮现"，增强深度感
- **文件：** `DetailPanel.tsx`

**T3：Repo Tabs 下划线滑动**
- **实现：** GSAP 驱动的绝对定位指示器，`x` + `width` 同时动画
- **参数：** 300ms `token-ease-out`，`willChange: transform, width`
- **效果：** Tab 切换时下划线平滑滑动，对齐精度 < 2px
- **文件：** `RepoTabs.tsx`

**T4：页面级入场动画**
- **TopBar：** 从顶部滑入（`y: -44 → 0`，150ms）
- **卡片：** stagger 淡入（`y: 8 → 0`，每张延迟 80ms）
- **StatusBar：** 从底部滑入（`y: 36 → 0`，150ms，延迟 400ms）
- **总时长：** ~550ms，LCP < 500ms
- **文件：** `TopBar.tsx`, `TaskGrid.tsx`, `StatusBar.tsx`

**T5：LED 呼吸去同步化**
- **实现：** CSS `animation-delay` 6 档不规则延迟（0s / 0.27s / 0.53s / 0.79s / 1.13s / 1.41s）
- **TopBar LED：** 固定延迟（`data-led-delay="1"` 和 `"2"`）
- **卡片 LED：** `useMemo` 随机分配（避免重渲染时动画重启）
- **效果：** 多个 LED 不再同步呼吸，700ms 后相位完全错开
- **文件：** `index.css`, `TopBar.tsx`, `StatusLed.tsx`

## 技术亮点

### 1. 字体配对策略

**Inter Tight Variable：**
- Condensed sans，适合 Dashboard 高信息密度
- Variable font，一次加载支持 300-900 weight
- 用于需要突出的文字（品牌、标题、状态）

**JetBrains Mono：**
- 保留用于数值、路径、时间戳
- 维持终端美学（atmospheric 流派）

**对比效果：**
- 修复前：全是等宽字体，层级依赖字重
- 修复后：字体对比+字重，视觉层级更清晰

### 2. GSAP 动效预算

| 动效 | 类型 | Duration | Stagger/Delay | GPU 加速 |
|------|------|----------|---------------|----------|
| TopBar 滑入 | transform (y) | 150ms | 0ms | ✅ |
| 卡片入场 | transform (y) + opacity | 300ms | 80ms stagger | ✅ |
| StatusBar 滑入 | transform (y) | 150ms | 400ms delay | ✅ |
| 面板打开 | transform (y, scale) | 260ms | 0ms | ✅ |
| Tab 下划线 | transform (x) + width | 300ms | 0ms | ⚠️ (width 触发 layout) |
| LED 呼吸 | opacity (CSS) | 2000ms | 0-1.41s delay | ✅ |

**总动画点数：** 5 个（符合 3-5 个预算）  
**总首屏时长：** ~550ms（符合 ≤ 1.2s 预算）

### 3. 不规则延迟算法

**LED 延迟值：** 0s / 0.27s / 0.53s / 0.79s / 1.13s / 1.41s

**选择原则：**
- 不是周期（2s）的整数倍 → 永不重新同步
- 质数或不规则间隔 → 打破机械感
- 覆盖 0–2s 范围 → 均匀分布

**实现细节：**
- TopBar LED：固定延迟（位置固定）
- 卡片 LED：`useMemo(() => random(1-6), [])` 稳定随机（避免重渲染时动画重启）
