## 背景

当前 `acp-client.mjs` 和 `task-recorder.mjs` 的错误处理比较简单，只抛出 `err.message`，缺少上下文信息。AI agent（Claude/Grok）调试时看到错误消息，不知道：
- 发生在哪个 JSON-RPC 方法
- 当时 sessionId 是什么
- 进程/文件处于什么状态
- 如何排查

**PRD 路径：** `工程师任务/2026-07-04-0340-AI友好化-测试与文档/PRD.md`

**关键决策（来自 PRD §2.4）：**
- **错误信息三要素：** 发生了什么 + 当时状态 + 如何排查
- **不改变抛出点** — 只改 `throw new Error(...)` 的消息内容，不改 catch 逻辑

---

## 只允许修改这些文件

1. **修改** `src/acp-client.mjs` — 改进约 10 个 catch 块的错误消息
2. **修改** `src/task-recorder.mjs` — 改进约 5 个 catch 块的错误消息

**禁止**改动其他任何文件。不要改 try 块的逻辑，不要改 catch 后的控制流（如 `process.exit`），只改 `throw new Error(...)` 或 `console.error(...)` 的消息文本。

## 改进原则

### 三要素模板

```js
throw new Error(
  `<动作> failed: ${err.message}\n` +
  `<关键状态 1>: ${value1}\n` +
  `<关键状态 2>: ${value2}\n` +
  `Hint: <如何排查>`
);
```

### 反例与正例

**❌ 反例（当前代码）：**
```js
} catch (err) {
  throw new Error(`JSON-RPC error: ${err.message}`);
}
```

**✅ 正例（改进后）：**
```js
} catch (err) {
  throw new Error(
    `JSON-RPC ${method} failed: ${err.message}\n` +
    `Session: ${this.sessionId || 'not initialized'}\n` +
    `State: ${this.state}\n` +
    `Hint: Check if Grok CLI is running and responsive.`
  );
}
```

## 修改一：`src/acp-client.mjs` — 改进 JSON-RPC 错误

找到以下 catch 块并改进（可直接采用）：

### 1. `request()` 方法的超时错误

**原代码（约第 80-90 行）：**
```js
const timer = setTimeout(() => {
  reject(new Error(`JSON-RPC timeout after ${timeout}ms`));
}, timeout);
```

**改为：**
```js
const timer = setTimeout(() => {
  reject(new Error(
    `JSON-RPC ${method} timed out after ${timeout}ms\n` +
    `Session: ${this.sessionId || 'not initialized'}\n` +
    `Params: ${JSON.stringify(params).slice(0, 100)}...\n` +
    `Hint: Try increasing --timeout-ms or check if Grok CLI is stuck.`
  ));
}, timeout);
```

### 2. `request()` 方法的通用错误

**原代码（约第 100 行）：**
```js
} catch (err) {
  throw new Error(`JSON-RPC error: ${err.message}`);
}
```

**改为：**
```js
} catch (err) {
  throw new Error(
    `JSON-RPC ${method} failed: ${err.message}\n` +
    `Session: ${this.sessionId || 'not initialized'}\n` +
    `Request ID: ${id}\n` +
    `Hint: Check Grok CLI logs (--debug-file) for protocol errors.`
  );
}
```

### 3. `handleLine()` 方法的 JSON 解析错误

**原代码（约第 120 行）：**
```js
} catch (err) {
  console.error(`grokACP: failed to parse line: ${line.slice(0, 100)}`);
  return;
}
```

**改为：**
```js
} catch (err) {
  console.error(
    `grokACP: JSON parse failed: ${err.message}\n` +
    `Line (first 100 chars): ${line.slice(0, 100)}\n` +
    `Session: ${this.sessionId || 'none'}\n` +
    `Hint: Grok CLI may have printed non-JSON to stdout (debug mode enabled?).`
  );
  return;
}
```

### 4. `runPrompt()` 方法的 spawn 错误

**原代码（约第 200 行）：**
```js
} catch (err) {
  throw new Error(`Failed to spawn grok agent stdio: ${err.message}`);
}
```

**改为：**
```js
} catch (err) {
  throw new Error(
    `Failed to spawn grok agent stdio: ${err.message}\n` +
    `Binary: ${this.grokBin}\n` +
    `Model: ${this.model}\n` +
    `Hint: Run 'grok --version' to verify Grok CLI is installed and in PATH.`
  );
}
```

### 5. 进程退出错误

**原代码（约第 210 行）：**
```js
proc.on("exit", (code, signal) => {
  this.reject?.(new Error(`grok agent stdio exited code=${code} signal=${signal}`));
});
```

**改为：**
```js
proc.on("exit", (code, signal) => {
  this.reject?.(new Error(
    `grok agent stdio exited unexpectedly\n` +
    `Exit code: ${code}, Signal: ${signal}\n` +
    `Session: ${this.sessionId || 'not initialized'}\n` +
    `Model: ${this.model}\n` +
    `Hint: Check stderr for Grok CLI errors, or try --debug.`
  ));
});
```

**原代码（约第 30 行）：**
```js

}
```

```js
function warnOnce(key, err) {
  if (warned.has(key)) return;
  warned.add(key);
  console.error(`grokACP [${key}]: ${err.message || err}`);
}
```
